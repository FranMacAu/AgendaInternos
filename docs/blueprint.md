# **App Name**: ContactVault

## Core Features:

- Display Contact List: Show a paginated list of all stored contacts, including their name, phone, service, and schedule.
- Add New Contact: Provide a form to add a new contact with fields for Name, Phone, Service, and Schedule. Upon submission, store the data.
- Edit Contact Details: Allow users to modify the details of existing contacts through an interactive form.
- Search and Filter Contacts: Implement a search bar that allows filtering contacts by any of the contact fields (Name, Phone, Service, Schedule).
- Delete Contact: Provide functionality to remove a contact from the agenda permanently.
- Firestore Data Persistence: Securely store and retrieve all contact information using Firebase's Firestore database.

## Style Guidelines:

- Primary color: A solid, trustworthy blue (#2D6CD1) for buttons, active states, and headings, conveying reliability.
- Background color: A very light, desaturated blue-grey (#F2F6FB) providing a clean and calm canvas.
- Accent color: A subdued teal-blue (#1D808F) to provide contrast and highlight interactive elements subtly, harmonizing with the primary blue.
- Body and headline font: 'Inter' (sans-serif) for its high readability and modern, neutral appearance, suitable for data-dense applications.
- Use clear and universally recognizable sans-serif icons for actions like 'add', 'edit', 'search', and 'delete', maintaining a minimalist aesthetic.
- Employ a responsive, card-based layout for contact listings, ensuring legibility and usability across various screen sizes. Forms should be clean and well-organized.
- Incorporate subtle transitions for form submissions, contact additions/edits, and search results to provide smooth user feedback without distraction.
